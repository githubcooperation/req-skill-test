## 亮点

1. **路径说明块统一且明确**：五个子 skill 均在第一个引用外部文件的步骤中使用 blockquote 块声明"所有路径相对于 skill 根目录（req-skill/）而非子目录"，有效消除 LLM 拼接错误路径的歧义。
2. **TBD 处理覆盖完整**：`question-strategy.md` 专门有"TBD 的表述识别"小节，列举 10+ 种用户可能说出的待定表达；req-complete、req-testcase、req-flowdoc 均有对 TBD 标注格式和后续处理的说明，形成完整闭环。
3. **测试用例质量约束具体且可执行**：req-testcase 的"质量自检"列出 9 条可量化检查项，并用"✗/✓"对照示例区分模糊描述与具体描述，降低了 LLM 生成低质量用例的风险。
4. **已有文件处理机制接近一致**：req-wireframe 和 req-flowdoc 均提供 A/B/C/D 四选项菜单，选项数量和措辞高度统一。
5. **截图失败的降级路径完整**：req-wireframe 步骤 3d 对截图失败有两次重试判断逻辑，两次均失败后给出具体人工兜底指引，体验设计周全。
