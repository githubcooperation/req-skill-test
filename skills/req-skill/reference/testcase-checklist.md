# 测试用例类型清单

## 正向流程测试

**适用场景**：所有有用户操作路径的功能模块。

验证在标准输入、正常条件下，功能是否按预期工作。每个功能的"阳光路径"都需要至少一个正向用例。

**最小示例**：
```
标题：用户使用有效手机号注册成功
前置条件：手机号未被注册
步骤：
  1. 进入注册页面
  2. 输入手机号 13800138000
  3. 点击获取验证码
  4. 输入收到的6位验证码
  5. 设置密码 Test@1234
  6. 点击注册
预期结果：注册成功，跳转到首页，显示欢迎信息
```

## 异常流程测试

**适用场景**：所有有输入或外部依赖的功能模块。

验证在异常条件下（错误输入、服务不可用、网络中断等），系统是否能正确处理并给出合理的反馈。

**最小示例**：
```
标题：使用已注册手机号再次注册
前置条件：手机号 13800138000 已被注册
步骤：
  1. 进入注册页面
  2. 输入手机号 13800138000
  3. 点击获取验证码
预期结果：提示"该手机号已注册，请直接登录"，提供登录入口链接
```

## 边界值测试

**适用场景**：涉及数值范围、长度限制、数量限制的功能。

测试在边界点和边界点附近的输入下系统的行为。每个边界至少覆盖四个点：低于下限、下限值、上限值、超过上限。

**最小示例**：
```
标题：密码长度恰好为最小值（8位）
前置条件：注册页面
步骤：输入密码 Aa1@5678（恰好8位）
预期结果：密码校验通过

标题：密码长度低于最小值（7位）
前置条件：注册页面
步骤：输入密码 Aa1@567（仅7位）
预期结果：提示"密码长度不能少于8位"
```

## 状态流转测试

**适用场景**：涉及状态机的实体（订单、工单、审批等）。

验证每个合法的状态转换是否正确执行，每个非法的状态转换是否被阻止。

**最小示例**：
```
标题：已支付订单成功转为已发货
前置条件：存在一个"已支付"状态的订单
步骤：商家点击"发货"按钮，填写快递信息
预期结果：订单状态变为"已发货"，用户收到发货通知

标题：已取消订单不能再次发货
前置条件：存在一个"已取消"状态的订单
步骤：商家尝试点击"发货"按钮
预期结果：按钮不可点击或点击后提示"该订单已取消，无法发货"
```

## 权限测试

**适用场景**：涉及多角色或权限控制的功能。

验证每个角色只能执行其权限范围内的操作，不能越权访问或操作。

**最小示例**：
```
标题：普通用户无法访问管理后台
前置条件：以普通用户身份登录
步骤：直接访问管理后台 URL /admin/dashboard
预期结果：返回403或跳转到无权限提示页面
```

## 输入校验测试

**适用场景**：所有有表单输入的功能。

验证各种非法输入是否被正确拦截，包括：空值、特殊字符、超长输入、SQL 注入、XSS 脚本、格式错误等。

**最小示例**：
```
标题：手机号输入包含字母
步骤：在手机号输入框输入 138abc00000
预期结果：提示"请输入正确的手机号格式"

标题：输入框插入 XSS 脚本
步骤：在昵称输入框输入 <script>alert('xss')</script>
预期结果：脚本被转义或过滤，不执行，正常保存为纯文本
```

## 并发测试

**适用场景**：涉及共享资源操作（库存扣减、余额变更、名额抢占等）的功能。

验证多个用户或请求同时操作同一资源时的数据一致性。

**最小示例**：
```
标题：两个用户同时抢购最后一件库存
前置条件：商品库存为1
步骤：用户A和用户B几乎同时点击购买
预期结果：只有一个用户购买成功，另一个提示"库存不足"；最终库存为0，不会出现负数
```

## 幂等性测试

**适用场景**：涉及支付、提交、确认等关键操作。

验证重复提交同一请求时不会导致重复处理（重复扣款、重复创建等）。

**最小示例**：
```
标题：用户快速双击支付按钮
前置条件：订单处于待支付状态
步骤：用户快速连续点击两次支付按钮
预期结果：只产生一笔支付记录，不会重复扣款
```

## 数据完整性测试

**适用场景**：涉及数据创建、修改、删除的功能。

验证数据操作后关联数据的一致性（如删除分类时该分类下的商品如何处理）。

**最小示例**：
```
标题：删除分类后其下商品的处理
前置条件：分类"电子产品"下有5个商品
步骤：管理员删除分类"电子产品"
预期结果：提示"该分类下有5个商品，请先迁移商品后再删除"，不允许直接删除
```

## 性能基准测试

**适用场景**：需求文档中有明确性能指标的功能。

如果需求中没有提及性能指标，此类型不适用，不生成对应用例。

**最小示例**：
```
标题：商品列表页加载时间
前置条件：数据库中有10000个商品
步骤：访问商品列表页首页
预期结果：页面完全加载时间不超过2秒（需求要求≤2s）
```

## 兼容性测试

**适用场景**：需求文档中有明确兼容性要求的功能（如指定支持的浏览器、设备、操作系统）。

如果需求中没有提及兼容性要求，以 P2 级别记录"建议补充兼容性要求"，但不生成具体用例。

**最小示例**：
```
标题：在 iOS Safari 中完成注册流程
前置条件：使用 iPhone（iOS 16+），Safari 浏览器
步骤：完成完整的注册流程
预期结果：所有步骤正常执行，无样式错乱或功能缺失
```

## 回退与撤销测试

**适用场景**：涉及可逆操作的功能（取消订单、撤回消息、回退审批等）。

**最小示例**：
```
标题：用户取消已支付订单并退款
前置条件：订单已支付，在可取消时间内
步骤：用户点击"取消订单"
预期结果：订单状态变为"已取消"，退款按原支付方式发起，库存恢复
```

## 空数据/首次使用测试

**适用场景**：列表页、统计页、用户首次使用等场景。

**最小示例**：
```
标题：新用户首次查看订单列表
前置条件：新注册用户，无历史订单
步骤：进入"我的订单"页面
预期结果：显示空状态页，包含引导文案"还没有订单，去逛逛"及商城入口
```

## 移动端专项测试

**适用场景**：需求文档中提及移动端（App、H5、小程序）的功能。

如果需求明确面向移动端，需覆盖以下子类型（根据功能适用性选择）：

### 手势与交互

- 滑动操作（左滑删除、下拉刷新、上拉加载更多）
- 长按操作（长按进入编辑/多选模式）
- 双指缩放（图片查看、地图）
- 拖拽排序（列表项重排）

**最小示例**：
```
标题：订单列表左滑删除
前置条件：用户有一个待支付订单
步骤：在订单列表页对该订单执行左滑操作，点击"删除"
预期结果：订单从列表中移除，显示撤销提示（如"已删除，点击撤销"）
```

### 横竖屏切换

- 切换后布局是否正常（不重叠、不截断）
- 表单数据是否保留
- 弹窗/对话框状态是否保持

### 离线与弱网

- 无网络时打开页面：显示离线提示或缓存数据
- 弱网（2G/3G）下提交表单：超时处理和重试机制
- 网络恢复后数据同步

**最小示例**：
```
标题：离线状态下查看已加载的订单列表
前置条件：用户在联网状态下已打开过订单列表页
步骤：关闭网络，重新进入订单列表页
预期结果：显示上次加载的缓存数据，顶部提示"当前处于离线状态，数据可能不是最新的"
```

### 推送通知

- 前台收到通知的展示形式
- 点击通知跳转到正确页面
- 通知权限被拒绝时的降级处理
- 角标（badge）数字更新

### 系统中断与恢复

- 来电中断操作后恢复
- 切换到其他 App 再返回：数据保留、页面状态保持
- 系统杀掉进程后重新打开：登录态保持、草稿恢复
