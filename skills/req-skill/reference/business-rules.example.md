## 数据校验规则

<!-- 基于 raw-materials 整理，来源：会议记录、用户故事草稿 -->
- 手机号：仅支持中国大陆手机号（1开头，11位数字）
- 任务标题：必填，建议最大长度 200 字符
- 截止日期：可选，若填写则必须是未来日期
- 任务优先级：枚举值（紧急/高/中/低），不可自定义
- 密码策略：参见 `reference/security-best-practices.md` 通用规范


## 状态流转规则

<!-- 来源：feature-priority-draft.md P0 功能 -->
- 任务：待开始 → 进行中（条件：负责人开始操作）
- 任务：进行中 → 已完成（条件：负责人标记完成）
- 任务：已完成 → 进行中（条件：管理员重新打开）
- 任务：任意状态 → 已关闭（条件：管理员关闭，可由已完成关闭）
- 任务：已关闭 → 不可恢复（只读归档）
- **注意**：子任务状态与父任务独立流转，父任务不自动随子任务完成


## 权限矩阵

<!-- 来源：meeting-notes-20260318.md + stakeholder-feedback-raw.txt（张敏/李强要求权限隔离） -->

| 操作 | 项目管理员 | 项目成员 | 只读角色 |
|------|-----------|---------|---------|
| 查看本项目任务 | ✓ | ✓ | ✓ |
| 创建/编辑任务 | ✓ | ✓ | ✗ |
| 删除任务 | ✓ | ✗（仅自己创建的） | ✗ |
| 管理项目成员 | ✓ | ✗ | ✗ |
| 查看其他项目 | ✗（项目间隔离） | ✗ | ✗ |
| 导出数据 | ✓ | ✗ | ✗ |

**权限隔离要求（强制）**：不同项目间数据完全隔离，成员无法看到未加入项目的任务（来源：张敏、李强反馈）

## 业务数值参数

<!-- 来源：feature-priority-draft.md + stakeholder-feedback-raw.txt -->
- 任务状态数量：4 个（待开始/进行中/已完成/已关闭）
- 子任务层级：支持至少 2 级（父任务 + 子任务）
- 截止日提醒提前量：N 小时（N 可配置，具体值 TBD）
- 每日推送时间：早 09:00（固定，来源：李强 feedback）
- 数据导出格式：Excel / CSV（待研发确认，当前优先级争议中）
- 附件支持类型：图片、文档（P1 功能，具体格式 TBD）


## 项目特有规则

<!-- 来源：meeting-notes-20260318.md + stakeholder-feedback-raw.txt -->
- **UI 简洁原则**：界面需简洁，避免过多字段，过度复杂导致用户弃用（赵磊/王芳 反馈）
- **客服与工单隔离**：客服工单系统与任务管理系统是两个独立产品，不得在本系统中混入工单功能（王芳明确要求）
- **任务模板**：客服/运营场景高频需要模板创建子任务，P1 优先实现（张敏/王芳 feedback）
- **避免 Jira 复杂度**：字段设计保持简洁，不要堆砌可选字段（刘伟/李强/王芳 反馈）
- **排除功能**：考勤管理、报销单关联、CRM 客户管理功能本身，明确不在本系统范围内
