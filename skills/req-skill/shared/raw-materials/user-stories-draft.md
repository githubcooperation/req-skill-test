## Epic 1：任务基础管理

---

**US-001** 创建任务
> 作为 **项目成员**，
> 我希望能 **快速创建一个任务并指定负责人和截止日期**，
> 以便 **让团队成员清楚自己的工作职责和时间要求**。

验收标准：
- [ ] 创建任务时可填写：标题（必填）、描述、负责人、截止日期、优先级
- [ ] 标题为空时无法提交，有错误提示
- [ ] 创建成功后，负责人收到系统通知
- [ ] 整个创建流程不超过 3 步

---

**US-002** 更新任务状态
> 作为 **任务负责人**，
> 我希望能 **一键更新任务的当前状态**，
> 以便 **让项目相关人实时了解进展，不需要反复被追问**。

验收标准：
- [ ] 状态选项：待开始 / 进行中 / 已完成 / 已阻塞 / 已取消
- [ ] 状态变更后，任务关注人/创建人收到通知
- [ ] 状态变更记录在任务动态日志中
- [ ] 移动端同样可以更新状态

---

**US-003** 查看"我的任务"
> 作为 **销售人员**，
> 我希望能 **看到一个聚合了我所有任务的视图，并按今天/本周/全部分组**，
> 以便 **每天开始工作时快速知道今天要做什么，不用在多个项目间切换**。

验收标准：
- [ ] 默认显示"今日待办"（截止日期为今天 + 即将超时）
- [ ] 可切换到"本周"和"全部"视图
- [ ] 各任务显示：标题、所属项目、截止时间、优先级
- [ ] 可以直接在这个视图里更新任务状态

## Epic 2：团队协作

---

**US-004** 任务评论与@通知
> 作为 **运营人员**，
> 我希望能 **在任务里直接评论并@相关人**，
> 以便 **跨部门沟通在任务上下文中进行，不需要跳到微信或飞书再单独联系**。

验收标准：
- [ ] 支持在任务详情页发表评论
- [ ] 评论中 @某人，该人收到通知，通知包含任务链接
- [ ] 支持回复某条评论
- [ ] 评论支持上传图片附件
- [ ] 评论不可删除（只能由管理员操作）

---

**US-005** 任务模板
> 作为 **客服主管**，
> 我希望能 **用预设模板快速创建包含子任务的任务包**，
> 以便 **处理标准化流程（如客诉处理）时不需要每次手动重建相同的子任务**。

验收标准：
- [ ] 项目管理员可以创建和管理任务模板
- [ ] 模板可包含：主任务 + 多个子任务（含默认负责人角色、建议时长）
- [ ] 使用模板创建任务时，可以在生成前预览和修改
- [ ] 模板列表支持搜索

## Epic 3：通知与提醒

---

**US-006** 截止日期提醒
> 作为 **任务负责人**，
> 我希望能 **在任务截止前收到提前提醒**，
> 以便 **不会因为忘记而错过 deadline**。

验收标准：
- [ ] 默认在截止前 24 小时发送提醒
- [ ] 用户可自定义提醒时间（1小时/3小时/1天/2天）
- [ ] 提醒可以通过飞书消息 / 系统内消息发送（至少一种）
- [ ] 已完成的任务不再发送提醒

---

**US-007** 每日任务推送
> 作为 **销售人员**，
> 我希望能 **每天早上收到一条今日待办任务汇总**，
> 以便 **开始工作前就清楚当天的重点，而不是登录后手动找**。

验收标准：
- [ ] 推送时间默认 09:00，用户可自定义
- [ ] 内容：今日到期任务数量 + 任务列表（最多显示10条）
- [ ] 点击通知可直接跳转到"我的任务-今日"视图
- [ ] 用户可选择关闭每日推送

---

## Epic 4：权限管理

---

**US-008** 项目权限隔离
> 作为 **运营负责人**，
> 我希望能 **确保不同项目的任务数据互相不可见**，
> 以便 **竞品分析、内部策划等敏感内容不会被其他部门看到**。

验收标准：
- [ ] 每个项目有独立的成员列表
- [ ] 非项目成员无法搜索到、访问该项目的任何任务
- [ ] 项目管理员可以邀请 / 移除成员
- [ ] 系统管理员可查看所有项目（可配置是否开启）
