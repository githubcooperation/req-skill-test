# 格式校验与修复

> 📁 **路径速查**：skill 内部文件（shared/ 等）相对于 `req-skill/`；output 文件相对于 workspace 根目录；用 `Glob("**/<filename>")` 定位 shared 文件最可靠。

## 职责

读取 PRD 文档，检查并修复以下格式问题：
- `#### 3.2.N` 标题格式缺失
- `{低保真原型图}` 占位符缺失或数量不足
- `<!-- SLOT: -->` 残留标记未替换

**原则**：只管格式，不管内容。不修改原文档的业务描述和语义。

**工具**：使用 Edit（局部替换）和 Bash（grep 验证）。不要用 Write 重写整个文档。

---

## 执行时机

- **自动执行**：作为 req-complete 第五步输出环节的一部分，在写 changelog 之前执行
- **单独触发**：用户说"请对 output/prd-updated.md 进行格式校验"时触发

---

## 输入

`output/prd-updated.md`

---

## 执行流程

### 第 1 步：读取文件

读取 `output/prd-updated.md`。

### 第 2 步：分析文档结构

扫描文档，回答以下三个问题：

1. **标题层级**：文档是否使用 `#### 3.2.N` 格式的功能模块标题？还是使用了其他格式（如 `###`、`##`）？
2. **占位符**：文档中是否包含 `{低保真原型图}` 字面文本？有多少个？
3. **卡片数量**：文档中描述了多少个功能区块（页面/表单/弹窗/列表等）？

### 第 3 步：识别每个功能区块

对每个功能区块（通常对应 `##` 或 `###` 下的一个内容段落）：

1. 判断该区块的功能内容：它包含哪些界面（列表页、详情页、表单页、弹窗等）
2. 根据内容判断该区块需要几个 `{低保真原型图}`：
   - 只有 1 个界面（单一页面） → 1 个占位符
   - 有多个 Tab/状态/弹窗 → 每个 Tab/状态/弹窗各 1 个
   - 有子章节描述独立的弹窗/表单 → 额外 1 个

### 第 4 步：执行修复

对每个功能区块执行以下修复操作：

#### 4a. 修复标题格式

如果区块标题不是 `#### 3.2.N` 格式：

**不要修改原始标题名称**，只需将标题层级调整为 `#### 3.2.{序号}` 格式。

示例：
- 原始：`### 注册流程` → 修复为 `#### 3.2.1 注册流程`
- 原始：`### 课程购买` → 修复为 `#### 3.2.5 课程购买`

**注意**：如果文档已经使用了 `#### 3.2.N` 格式，则跳过此步骤。

#### 4b. 插入 {低保真原型图}

对每个功能区块，检查是否已有 `{低保真原型图}`：

- 如果区块已经有足够数量的占位符 → 跳过
- 如果缺少占位符 → 在适当位置插入

**插入位置规则**：

- 单一界面区块：在区块末尾（`---` 分隔线之前，或下一个同级/更高级标题之前）插入
- 多界面区块（有 `**加粗行**` 标记 Tab/状态）：
  - 每个 `**...**` 加粗行下方插入一个 `{低保真原型图}`
  - 加粗行之后如果已有 `{低保真原型图}`，不要重复插入
- 多界面区块（无加粗行但有明确子界面描述）：
  - 在每个子界面描述段落的末尾插入

**示例**：

```
#### 3.2.3 优惠券

（区块内容...）

**[使用中]**

{低保真原型图}

**[未使用]**

{低保真原型图}

---
```

#### 4c. 验证修复结果

修复完成后，用 Bash 运行：

```bash
echo "占位符数量: $(grep -c '{低保真原型图}' output/prd-updated.md)"
echo "3.2.N 标题数: $(grep -c '#### 3\.2\.' output/prd-updated.md)"
```

确认：
1. 占位符数量 ≥ 功能区块数量
2. 所有功能区块都有 `#### 3.2.N` 标题
3. 文件中无残留的 `<!-- SLOT:` 标记

如果验证不通过，继续修复直到通过。

### 第 5 步：报告结果

在对话中输出修复报告：

```
格式校验完成！

| 检查项 | 结果 |
|------|------|
| 功能区块标题格式 | ✅ / ❌ |
| {低保真原型图} 占位符 | ✅ N 个（共 N 个）/ ❌ N 个缺失（已补 N 个） |
| 修复详情 | {具体修复了哪些内容} |
```

## 边界情况处理

### 文档完全没有功能区块标题

如果文档中没有 `###`、`####`、`##` 标题（或只有 `#` 一级标题）：

```
output/prd-updated.md 中未检测到功能区块（### 或 #### 标题）。
这可能是一个只有 1-2 个 ## 简短章节的简短 PRD，而非完整的需求文档。
格式校验不适用于此文档。
```

### 文档已经完全合规

如果文档已经使用了 `#### 3.2.N` 格式且占位符数量正确：

```
output/prd-updated.md 格式完全合规，无需修复。
✅ 功能区块数: N 个
✅ 占位符数: N 个
✅ 无残留 SLOT 标记
```
