# 需求文档审查缺陷清单

- docs/prd.md

## P0（阻塞性）- 共 5 个

### 1. 订单状态流转不完整
- **模块**: 课程购买 / 订单管理
- **类型**: incomplete_flow
- **描述**: 订单状态仅列出"待支付、已支付、已取消"，但未说明：
  - 状态之间的转换条件（如已支付订单在什么条件下变为已取消）
  - 是否存在其他状态（如已退款、已完成）
  - 支付超时后订单状态如何变化
- **影响**: 开发无法确定状态机实现，测试无法覆盖所有状态流转

### 2. 退款流程未说明
- **模块**: 课程购买 / 订单管理
- **类型**: missing
- **描述**: 文档提到"已支付的订单在一定条件下可以申请退款"，但未说明：
  - 退款的具体条件是什么（可退款的时间窗口？课程进度要求？）
  - 退款申请需要审核吗
  - 退款后订单状态如何变化
  - 退款是否退回原支付渠道
- **影响**: 核心交易功能不完整，无法开发测试

### 3. 支付流程缺少失败处理
- **模块**: 课程购买 / 购买流程
- **类型**: incomplete_flow
- **描述**: 文档说明"选择支付方式进行支付"，但未说明：
  - 支付失败后如何提示用户
  - 支付失败后订单状态如何处理
  - 是否支持重新支付
  - 支付超时如何处理
- **影响**: 支付是核心交易环节，失败场景必须明确

### 4. 验证码有效期未说明
- **模块**: 用户注册与登录 / 注册流程
- **类型**: missing_constraint
- **描述**: 注册流程提到"输入验证码"，但未说明验证码有效期是多久
- **影响**: 开发无法确定验证码存储和过期策略，测试无法验证过期行为

### 5. 密码复杂度要求未量化
- **模块**: 用户注册与登录 / 注册流程
- **类型**: missing_constraint
- **描述**: 文档要求"密码需要有一定的复杂度要求"，但未给出具体规则：
  - 最小长度
  - 是否需要大小写、数字、特殊字符
  - 最大长度限制
- **影响**: 开发无法实现，测试无法验证
